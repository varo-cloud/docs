This guide walks you through everything you need to make your first authenticated request to the Varo Cloud API — from generating a key to triggering a real deployment.

<Info>
  **Base URL:** `https://api.varocloud.io/v1` — all endpoints in this guide are relative to it.
</Info>

## Step 1: Generate an API Key

All requests to the Varo Cloud API are authenticated with a bearer token. To create one:

<Steps>
  <Step title="Open the dashboard">
    Sign in at [app.varocloud.io](https://app.varocloud.io) and navigate to **Settings → API Keys**.
  </Step>
  <Step title="Create a new key">
    Click **Create API Key**, give it a descriptive name (e.g. `local-dev` or `ci-pipeline`), and choose the scopes it needs.
  </Step>
  <Step title="Copy and store it securely">
    The full key is shown **only once**. Copy it immediately and store it in a secret manager or environment variable — never commit it to source control.

    ```bash
    export VARO_API_KEY="vk_live_..."
    ```
  </Step>
</Steps>

<Warning>
  Treat your API key like a password. If a key is ever exposed, rotate it immediately from the dashboard.
</Warning>

## Step 2: Make Your First Request

Let's verify your key works by listing your projects. Pass the key in the `Authorization` header as a bearer token:

<CodeGroup>

```bash cURL
curl https://api.varocloud.io/v1/projects \
  -H "Authorization: Bearer $VARO_API_KEY"
```

```javascript Node.js
const response = await fetch("https://api.varocloud.io/v1/projects", {
  headers: {
    Authorization: `Bearer ${process.env.VARO_API_KEY}`,
  },
});

const { data } = await response.json();
console.log(data);
```

```python Python
import os
import requests

response = requests.get(
    "https://api.varocloud.io/v1/projects",
    headers={"Authorization": f"Bearer {os.environ['VARO_API_KEY']}"},
)

print(response.json()["data"])
```

</CodeGroup>

A successful response returns your projects wrapped in the standard envelope:

```json
{
  "data": [
    {
      "id": "proj_a1b2c3",
      "name": "my-first-app",
      "created_at": "2026-06-15T10:30:00Z"
    }
  ],
  "pagination": { "total": 1, "per_page": 20, "next_cursor": null },
  "meta": { "request_id": "req_abc123" }
}
```

If you get back a `401 Unauthorized`, double-check that your key is correct and hasn't been revoked. See [Errors](/api-reference/errors) for the full list of response codes.

## Step 3: Trigger a Deployment

Now that authentication works, let's do something real: kick off a deployment. Replace `proj_a1b2c3` with an ID from the response above.

<CodeGroup>

```bash cURL
curl -X POST https://api.varocloud.io/v1/deployments \
  -H "Authorization: Bearer $VARO_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "project_id": "proj_a1b2c3",
    "environment": "production",
    "git_ref": "main"
  }'
```

```javascript Node.js
const response = await fetch("https://api.varocloud.io/v1/deployments", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.VARO_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    project_id: "proj_a1b2c3",
    environment: "production",
    git_ref: "main",
  }),
});

const { data: deployment } = await response.json();
console.log(`Deployment ${deployment.id} status: ${deployment.status}`);
```

```python Python
import os
import requests

response = requests.post(
    "https://api.varocloud.io/v1/deployments",
    headers={
        "Authorization": f"Bearer {os.environ['VARO_API_KEY']}",
        "Content-Type": "application/json",
    },
    json={
        "project_id": "proj_a1b2c3",
        "environment": "production",
        "git_ref": "main",
    },
)

deployment = response.json()["data"]
print(f"Deployment {deployment['id']} status: {deployment['status']}")
```

</CodeGroup>

The response returns immediately with the newly created deployment in a `pending` state:

```json
{
  "data": {
    "id": "dep_x9y8z7",
    "project_id": "proj_a1b2c3",
    "environment": "production",
    "status": "pending",
    "git_ref": "main",
    "created_at": "2026-07-06T14:22:10Z"
  },
  "meta": { "request_id": "req_def456" }
}
```

## Step 4: Poll for Status

Deployments run asynchronously. Poll the deployment endpoint until `status` becomes `succeeded` or `failed`:

```bash
curl https://api.varocloud.io/v1/deployments/dep_x9y8z7 \
  -H "Authorization: Bearer $VARO_API_KEY"
```

<Tip>
  For production integrations, use [webhooks](/api-reference/webhooks) instead of polling. Varo Cloud will push signed events to your endpoint as soon as a deployment's status changes.
</Tip>

## Handling Errors

Every error response includes a machine-readable `code` and a human-readable `message`:

```json
{
  "error": {
    "code": "invalid_request",
    "message": "Field 'git_ref' is required.",
    "field": "git_ref"
  },
  "meta": { "request_id": "req_ghi789" }
}
```

Include the `meta.request_id` when contacting support — it lets us trace your request through our logs instantly.

## Rate Limits

The default limit is **1,000 requests per 10-minute window** per API key. Monitor these headers on every response:

| Header | Description |
| --- | --- |
| `X-RateLimit-Limit` | Requests allowed in the current window |
| `X-RateLimit-Remaining` | Requests remaining in the current window |
| `X-RateLimit-Reset` | Unix timestamp when the window resets |
