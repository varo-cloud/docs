This guide walks you through everything you need to make your first authenticated request to the Varo Cloud API — from generating a key to running a model and getting back a result.

<Info>
  **Base URL:** `https://inference.varo.cloud/v1` — all endpoints in this guide are relative to it.
</Info>

## Step 1: Generate an API Key

All requests to the Varo Cloud API are authenticated with a bearer token. To create one:

<Steps>
  <Step title="Open the dashboard">
    Sign in at [varo.cloud/auth](https://varo.cloud/auth) and open **API Keys** in your dashboard.
  </Step>
  <Step title="Create a new key">
    Click **Create Key** and give it a descriptive name (e.g. `local-dev` or `ci-pipeline`).
  </Step>
  <Step title="Copy and store it securely">
    The full key is shown **only once**. Copy it immediately and store it in a secret manager or environment variable — never commit it to source control.

    ```bash
    export VARO_API_KEY="sk_live_..."
    ```
  </Step>
</Steps>

<Warning>
  Treat your API key like a password. If a key is ever exposed, delete it from the dashboard and create a new one right away.
</Warning>

## Step 2: Create a Generation

Let's create an image. Send a `POST` request to `/v1/generations` with the model you want to run and its inputs. Pass your key in the `Authorization` header as a bearer token.

Generations run **asynchronously**: this request creates a task and returns immediately with an `id` and a `status`. You'll poll that `id` in the next step to get the finished asset.

<CodeGroup>

```bash cURL
curl -X POST https://inference.varo.cloud/v1/generations \
  -H "Authorization: Bearer $VARO_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "google-nano-banana-pro/text-to-image",
    "prompt": "a cat sitting on a windowsill, golden hour lighting, photorealistic",
    "aspect_ratio": "4:3",
    "output_format": "jpeg",
    "resolution": "1K"
  }'
```

```javascript Node.js
const response = await fetch("https://inference.varo.cloud/v1/generations", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.VARO_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    model: "google-nano-banana-pro/text-to-image",
    prompt: "a cat sitting on a windowsill, golden hour lighting, photorealistic",
    aspect_ratio: "4:3",
    output_format: "jpeg",
    resolution: "1K",
  }),
});

const generation = await response.json();
console.log(generation.id, generation.status);
```

```python Python
import os
import requests

response = requests.post(
    "https://inference.varo.cloud/v1/generations",
    headers={
        "Authorization": f"Bearer {os.environ['VARO_API_KEY']}",
        "Content-Type": "application/json",
    },
    json={
        "model": "google-nano-banana-pro/text-to-image",
        "prompt": "a cat sitting on a windowsill, golden hour lighting, photorealistic",
        "aspect_ratio": "4:3",
        "output_format": "jpeg",
        "resolution": "1K",
    },
)

generation = response.json()
print(generation["id"], generation["status"])
```

</CodeGroup>

## Step 3: Poll for the Result

Fetch the generation by its `id` and keep polling until its `status` becomes `completed` (or `succeeded`). If it comes back `failed`, stop and read the error message.

<CodeGroup>

```bash cURL
# Poll until status is completed or succeeded
while true; do
  response=$(curl -s https://inference.varo.cloud/v1/generations/$GENERATION_ID \
    -H "Authorization: Bearer $VARO_API_KEY")
  status=$(echo "$response" | jq -r '.status')
  if [ "$status" = "completed" ] || [ "$status" = "succeeded" ]; then
    echo "$response" | jq -r '.output.url'
    break
  fi
  if [ "$status" = "failed" ]; then
    echo "$response" | jq -r '.error.message // "Generation failed"' >&2
    exit 1
  fi
  sleep 5
done
```

```javascript Node.js
async function pollGeneration(id) {
  while (true) {
    const res = await fetch(
      `https://inference.varo.cloud/v1/generations/${id}`,
      { headers: { Authorization: `Bearer ${process.env.VARO_API_KEY}` } }
    );
    const generation = await res.json();

    if (["completed", "succeeded"].includes(generation.status)) {
      return generation.output.url;
    }
    if (generation.status === "failed") {
      throw new Error(generation.error?.message ?? "Generation failed");
    }
    await new Promise((r) => setTimeout(r, 5000));
  }
}

console.log(await pollGeneration(generation.id));
```

```python Python
import time

def poll_generation(generation_id):
    while True:
        res = requests.get(
            f"https://inference.varo.cloud/v1/generations/{generation_id}",
            headers={"Authorization": f"Bearer {os.environ['VARO_API_KEY']}"},
        )
        generation = res.json()

        if generation["status"] in ("completed", "succeeded"):
            return generation["output"]["url"]
        if generation["status"] == "failed":
            raise RuntimeError(generation.get("error", {}).get("message", "Generation failed"))
        time.sleep(5)

print(poll_generation(generation["id"]))
```

</CodeGroup>

<Tip>
  Poll every few seconds rather than in a tight loop. Image models usually finish in seconds; video models can take longer.
</Tip>

## Step 4: Read the Response

The API returns a **generation object**. When `status` is `completed`, `output.url` points to your finished asset:

```json
{
  "id": "9d481365-2b49-4e94-ac9b-c3d2db5b2374",
  "object": "generation",
  "status": "completed",
  "model": "google-nano-banana-pro/text-to-image",
  "created_at": 1783996705,
  "output": {
    "type": "image",
    "url": "https://cdn.example.com/generations/.../result.jpeg"
  },
  "usage": {
    "cost_usd": 0.04
  }
}
```

| Field | Description |
| --- | --- |
| `id` | Unique identifier for this generation |
| `status` | Generation state — `completed` when the asset is ready |
| `model` | The model that produced the output |
| `output.type` | `image` or `video`, depending on the model |
| `output.url` | Direct link to download the generated asset |
| `usage.cost_usd` | Cost of this generation, in USD (varies by model and parameters) |

<Warning>
  If you get back a `401 Unauthorized`, double-check that your key is correct, starts with `sk_live_`, and hasn't been revoked. See [Errors](/api-reference/errors) for the full list of response codes.
</Warning>

## Step 5: Try Another Model

The same endpoint runs every model on the platform — just change the `model` field and its inputs. Here's a video model instead of an image one:

```bash
curl -X POST https://inference.varo.cloud/v1/generations \
  -H "Authorization: Bearer $VARO_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "alibaba-wan-2-7/reference-to-video",
    "prompt": "the cat slowly turns to look at the camera"
  }'
```

The response has the same shape, with `output.type` set to `video`:

```json
{
  "id": "9d481365-2b49-4e94-ac9b-c3d2db5b2374",
  "object": "generation",
  "status": "completed",
  "model": "alibaba-wan-2-7/reference-to-video",
  "created_at": 1783996705,
  "output": {
    "type": "video",
    "url": "https://cdn.example.com/generations/.../result.mp4"
  },
  "usage": {
    "cost_usd": 0.5
  }
}
```

<Tip>
  One API, one key, one response shape for every model — swap the `model` field to move between image, video, and other modalities without changing your integration.
</Tip>
